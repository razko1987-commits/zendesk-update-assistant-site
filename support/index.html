Support

Get Help via GitHub Issues

For questions, bug reports, or feature requests, please use our GitHub Issues page. This helps us track and respond to all support requests efficiently.

Submit a Support Request

What to Include

When submitting a support request, please include:

  • Your Zendesk subdomain - This helps us understand your setup
  • Brief description - What issue are you experiencing or what feature are you requesting?
  • Steps to reproduce - If reporting a bug, what steps led to the issue?
  • Expected vs. actual behavior - What should happen vs. what actually happens?
  • Screenshots - If applicable, screenshots can help us understand the issue better

Response Times

We aim to respond to all support requests within 2-3 business days. Standard plan subscribers receive priority support with faster response times.

Common Questions

Before submitting a support request, you may want to check the FAQ section on the app's page, which covers many common questions about the app.

Feature Requests

We welcome feature requests! Please submit them through GitHub Issues with the "feature request" label. We review all suggestions and consider them for future updates.

Bug Reports

If you encounter a bug, please report it through GitHub Issues with as much detail as possible. This helps us identify and fix issues quickly.
